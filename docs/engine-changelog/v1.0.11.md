The design-review polish pass on the card-system redesign (v1.0.10). Three changed blocks, no new blocks — no fence seeding required this release.

- **Changed `styles-card-v2`**: the card's left accent returns to the LAYER's own map color (`--layer-accent`, with the redesign blue as fallback) — restoring the card-to-overlay color tie that predates the redesign; the 4e error/empty edge accents still override via `:has`. Idle compact cards (layer off / no result) show the full-size 16px/700 layer name again — the 4b kicker treatment applies only while a value line is visible. New `.card-nearest-dot` rule for the nearest-row identity dot.
- **Changed `card-helpers`**: expanders now default CLOSED everywhere — `renderOfficeGroup` opens only on an explicit `open: true`, person `details` only on `detailsOpen: true`, and the desktop-width auto-open (with the internal `cardDesktopWidth`) is removed. `renderNearestRows` items accept an optional `dotColor` (an identity dot ahead of the name — school-type colors and the like).
- **Changed `polygon-factory`**: new `opts.pill` — an explicit opt that promotes the primary field (label + value, e.g. "District 1") into the header identifier pill and drops it from the body stack. Inert unless a registration passes it.
- Consume-side scripts are unchanged.
- Fork adoption: none required — the accent/label changes are a visual restyle at bump, `pill`/`dotColor` are inert new opts until a fork's registrations use them, and the closed-expander default needs no test changes (a closed `<details>` still exposes its summary text to `innerText` assertions). Behavior note for card copy: any fork prose that assumed offices/committees start open on desktop is now wrong — they start closed fleet-wide by design decision (2026-07 design review).
